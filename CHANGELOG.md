## [1.1.0] - 2026-06-03

### Added

- Roles & Responsibilities documentation
- Role-Based Onboarding guide
- Product Owner onboarding path
- Scrum Master onboarding path
- Tech Lead onboarding path
- Backend Engineer onboarding path
- Frontend Engineer onboarding path
- QA Engineer onboarding path
- DevOps Engineer onboarding path
- Community Contributor onboarding path
- RACI-style responsibility definitions for ERP Plus delivery roles
- Taiga state automation documentation (#ready-for-test / #done)
- Community contribution workflow clarification
- Incident Report issue template
- Organization-wide contribution governance improvements

### Changed

- Refactored role definitions previously duplicated across System and Organization sections
- Improved governance documentation structure
- Improved onboarding navigation and discoverability
- Clarified internal vs external contributor responsibilities
- Clarified Product Delivery ownership model
- Clarified relationship between Taiga, GitHub, CI/CD and Releases
- Updated CONTRIBUTING guides to align with ERP Plus governance model
- Updated global Issue Templates to support internal and community contribution flows

### Removed

- Duplicated role descriptions distributed across multiple documents
- Redundant onboarding information spread across unrelated sections

### Fixed

- Documentation gaps for Product Owners, Scrum Masters and Tech Leads
- Missing responsibility definitions across delivery roles
- Inconsistent contributor workflow descriptions between repositories

## Release Notes

ERP Plus Documentation v1.1.0 introduces the organizational and delivery governance layer required for multi-role collaboration.

This release establishes:

- Product Ownership model
- Scrum responsibilities
- Engineering responsibilities
- QA responsibilities
- DevOps responsibilities
- Community contribution model
- Role-based onboarding paths
- Standardized Taiga ↔ GitHub workflow documentation

The documentation platform now supports onboarding for both technical and non-technical stakeholders while preserving architecture, governance, delivery, and operational consistency.
