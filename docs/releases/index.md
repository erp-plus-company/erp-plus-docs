# Releases

This section documents how ERP Plus Documentation versions are prepared, reviewed, tagged, and published.

---

## Purpose

Release management ensures that documentation changes are:

- traceable
- reviewable
- versioned
- reproducible

---

## Release Lifecycle

ERP Plus Documentation follows a release-based workflow:

```txt
Draft
  ↓
Review
  ↓
Release Candidate (RC)
  ↓
Version Tag
  ↓
Published Release
```

---

## Versioning Model

Documentation versions follow Semantic Versioning principles.

```txt
MAJOR.MINOR.PATCH
```

Examples:

```txt
1.0.0
1.1.0
1.2.3
2.0.0
```
