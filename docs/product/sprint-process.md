# Sprint Process

ERP Plus uses two-week sprints.

---

## Sprint Goal

A sprint represents a planning and delivery cycle.

Duration:

```txt
2 weeks
```

---

## Sprint Structure

```txt
Sprint Planning
      ↓
Development
      ↓
Testing
      ↓
Review
      ↓
Release
```

---

## Planning

During planning:

- User Stories are prioritized
- Tasks are created
- Sprint scope is defined

---

## Execution

Developers implement Tasks linked to User Stories.

All work must follow:

```txt
User Story
   ↓
Branch
   ↓
Pull Request
```

---

## Validation

Validation includes:

- tests
- reviews
- CI checks

---

## Completion

A sprint is complete when:

- committed User Stories are delivered
- testing is complete
- releases are prepared
